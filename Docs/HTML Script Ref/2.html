2. Callbacks

 These functions are called by the game into the script after certain events.  Callbacks ( initiated by remoteCall, startTimer, watchKey etc... ) are "broadcasted" at the destination. The destination receives the call and executes this in the context of all running scripts. This allows scripts to communicate, however all defined functions also need unique names or to be defined within a namespace to prevent "accidental" calls of a function from another script running at the same time  If any callback fails, the error message is only displayed if enableAlerts(1) was executed previously.

  1. onInitialize()
    Perform any initialization here, this is called right after the script is loaded.
     
  2. onRelease()
    Called before the script is released.
